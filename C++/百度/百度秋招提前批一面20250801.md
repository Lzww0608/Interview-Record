## 2.OSI有哪七层模型

+ **会话层**：建立、管理和终止不同设备上的应用程序之间的会话（或称对话）。它负责通信的同步，可以在数据流中插入“同步点”或“检查点”。如果网络通信中断，可以在最后一个同步点恢复，而不需要从头开始。这对于大文件传输等场景非常重要。**RPC**
+ **表示层**：数据的格式化、加密/解密、压缩/解压缩。它确保一个系统应用层发送的数据能被另一个系统的应用层理解。可以看作是网络的“翻译官”。例如，将ASCII码转换为EBCDIC码，或者处理多媒体文件的编码格式（JPEG, MPEG）。现代网络中，数据的加密（如TLS/SSL）通常也认为是在这一层（或跨越了5、6、7层）完成的。
+ **应用层**：为用户应用程序提供网络服务的接口。这是用户直接接触和感知的一层。它不提供核心网络功能，而是作为应用程序和网络之间的“桥梁”。它负责处理高级的API，如文件传输、电子邮件、Web访问等。我们程序员在进行网络编程时，直接打交道的就是这一层提供的功能。**HTTP/HTTPS (网页浏览), FTP (文件传输), SMTP/POP3/IMAP (电子邮件), DNS (域名解析), Telnet (远程登录)。**

## 15.了解过IP分片吗，解释一下（答上来一半吧）
当一个 IP 数据包的总大小（包括 IP 头部）超过路径中某段链路的 MTU （1500/576字节）时，若不进行处理，该数据包会被直接丢弃。IP 分片的作用就是将大数据包 “拆分” 成多个小于等于 MTU 的片段，确保其能通过所有链路。
+ DF标志位判断是否允许分片
+ 标识、MF标志、片偏移量

实际场景中更倾向于禁用分片（DF=1），并通过`PMTUD`确保数据包大小不超过路径MTU，以避免分片带来的问题。例如：TCP 默认启用 DF=1，通过`PMTUD`动态调整数据包大小，几乎不需要依赖分片。


## 16.TCP报文长度管理是怎么实现的
+ 基础限制：MSS 定义单报文最大数据长度，确保不触发 IP 分片（结合 PMTUD 动态调整）。
+ 接收方限制：rwnd 确保发送量不超过接收方缓冲区容量（流量控制）。
+ 网络限制：cwnd 确保发送量不超过网络承载能力（拥塞控制）。
+ 优化机制：Nagle 算法等减少小报文，提升传输效率。




## 17.UDP最长报文长度是多少（我回答的是65535，应该错的，真正有效长度是1472字节）
+ 以太网 MTU 为 1500 字节时：1500（IP包总长度） - 20（IP头） - 8（UDP头） = 1472字节 数据部分 。

## 19.vlan是什么（一点没有听过，没答上来）
VLAN（Virtual Local Area Network，虚拟局域网） 是一种通过网络技术（软件配置）在物理局域网（LAN）基础上，将设备逻辑划分为多个独立 “虚拟子网” 的技术。简单来说，它能让物理上分散的设备，像处于同一个局域网中一样通信，同时与其他虚拟子网隔离开。
### 作用
1. 隔离广播域：隔离广播包（如ARP请求）
2. 增强安全性：不同vlan的设备无法直接通信（需要通过路由器/三层交换机）
3. 灵活管理设备：根据业务需求而非物理位置来分割网络
### 实现原理
VLAN 通过在以太网帧中插入VLAN 标签（Tag） 实现，遵循IEEE 802.1Q 标准。

## 20.网络安全相关，DDoS攻击（只宽泛的解释了一下含义）
DDoS 攻击（分布式拒绝服务攻击，Distributed Denial of Service）是一种通过多个分布式攻击源向目标系统（如服务器、网站、网络设备等）发送大量恶意流量，耗尽目标资源（带宽、计算能力、内存等），最终导致目标无法正常提供服务的网络攻击方式。

1. SYN Flood（SYN 洪水攻击）
利用 TCP 协议三次握手的漏洞：攻击者向目标发送大量伪造的 “SYN 连接请求”（第一步），但不回应服务器的 “SYN-ACK”（第二步），导致服务器维持大量 “半连接” 状态，耗尽连接队列资源，无法接收新的合法连接。
2. UDP Flood（UDP 洪水攻击）
UDP 协议无需建立连接，攻击者向目标的任意 UDP 端口发送大量随机数据包（如垃圾数据），占用目标的带宽和处理能力，导致正常 UDP 服务（如 DNS、视频流）瘫痪。
3. HTTP Flood（HTTP 洪水攻击）
模拟正常用户的行为，向 Web 服务器发送大量 HTTP 请求（如反复刷新页面、提交表单），尤其针对需要复杂计算的接口（如登录、搜索），耗尽服务器的 CPU 和内存资源，导致网站无法打开。

防御手段：
+ 基础防护：配置防火墙限制异常流量（如限制单 IP 连接数）、关闭不必要的端口 / 服务（减少攻击面）；
+ 流量清洗：通过专业 DDoS 防护设备（如抗 D 防火墙）或服务商，识别并过滤恶意流量，仅放行合法请求；
+ 分布式架构：用 CDN（内容分发网络）分散流量、负载均衡器分担压力，避免单点故障；
+ 黑洞路由：极端情况下，暂时将目标 IP “拉黑”（黑洞），切断所有流量（代价是业务暂时中断，但可避免更严重的资源耗尽）；
+ 监控与预警：通过网络监控工具实时检测流量异常（如带宽突增、连接数暴增），提前响应。

## 27.解释线程与进程的区别（这个准备的很好，答的蛮顺利的）
本质区别：资源是否独立
1. 资源独立性：struct mm_struct *mm, struct files_struct *files, struct signal_struct *signal, uid/gid
2. 标识字段：
   + pid：内核分配给每个 task_struct 的唯一标识（内核视角的 “任务 ID”），无论是进程还是线程，都有唯一的 pid。
   + tgid：线程组 ID，标识该任务所属的 “线程组”（即用户视角的 “进程”）。
3. 调度上下文：
  + 内核栈（void *stack）：每个 task_struct 都有独立的内核栈（用于内核态执行），进程和线程的内核栈互不共享。
  + 用户栈：
      + 进程的用户栈在自身独立的地址空间中；
      + 线程的用户栈虽然在共享的地址空间中，但每个线程有独立的栈内存区域（避免栈数据冲突）。
4. 生命周期：task_struct 中的 parent（父任务）、children（子任务链表）和 group_leader（线程组领头进程）字段描述了任务间的关系。
**当线程组领头进程终止时，整个线程组的所有线程都会被终止（除非设置了 CLONE_DETACHED 标志）。**
